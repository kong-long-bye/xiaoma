# Transformer-SVR/KNN 实验汇报与代码交接说明

## 1. 研究目标与当前结论

原论文方法为：

```text
WiFi WAP 信号 → Transformer 编码器 → SVR → 经纬度
```

本项目没有拿到原论文最终代码，因此先基于当前可运行代码复现，再在不修改
Transformer 结构的前提下改进下游定位方法。

正式 Evaluation 结果如下：

| 方法 | 平均误差（米） | 中位误差（米） | 定位 |
|---|---:|---:|---|
| 特征融合 Transformer-SVR | 6.5943 | 4.9578 | 保留 SVR 的改进 |
| Transformer-KNN | 6.1052 | 4.3592 | 当前最佳下游回归头 |

## 2. 特征融合 Transformer-SVR 如何改进

Building 0 的 520 个 WAP 中，有大量特征几乎始终缺失。以训练集缺失率
0.98 为阈值过滤后，保留 98 个 WAP。缺失标记 100 被替换为 -105，再使用
训练阶段保存的 scaler 做缩放。

原始 Transformer-SVR 只把 Transformer 压缩后的 64 维 latent 输入 SVR。
改进方法同时保留 Transformer latent 和过滤后的 RSSI：

```text
98维过滤RSSI ──→ 冻结Transformer ──→ 64维latent ──┐
       └──────────────────────→ 98维缩放RSSI ─────┤
                                                  ↓
                                     拼接为162维融合特征
                                                  ↓
                                   MultiOutput RBF-SVR
                                                  ↓
                                          经纬度坐标
```

Transformer latent 表示全局、非线性的高层特征；原始过滤 RSSI 保留局部信号
强弱和接入点细节。二者拼接可以补偿 Transformer 压缩过程中丢失的定位细节。

参数只在 trainingData.csv 上用位置分组两折交叉验证选择：

```text
C: 50, 100, 200
epsilon: 0.1, 0.3, 0.5
kernel: rbf
gamma: scale
目标标准化: 关闭
选择目标: distance_mean + 0.5 * distance_median
```

训练集交叉验证最终选择：

```text
C = 200
epsilon = 0.5
gamma = scale
```

官方 Evaluation（Building 0，536 条）结果：

```text
distance_mean   = 6.5942772084
distance_median = 4.9578234008
distance_p90    = 13.6221397059
distance_max    = 34.8985134595
```

## 3. Transformer-KNN 如何改进

Transformer-KNN 不使用原始 RSSI 直接定位，而是读取同一个冻结 Transformer
产生的 latent，再用距离加权 KNN 回归坐标：

```text
98维过滤RSSI → 冻结Transformer → 64维latent → 加权KNN → 经纬度
```

KNN 参数只使用 trainingData.csv 的 PHONEID 分组两折交叉验证选择：

```text
k = 10
metric = euclidean
weights = distance
latent标准化 = false
```

官方 Evaluation 结果：

```text
distance_mean   = 6.1052337996
distance_median = 4.3591755311
distance_p90    = 13.3249970269
distance_max    = 35.1039875877
```

该方法不修改 Transformer，但把下游回归头从 SVR 换成 KNN，因此论文中应称为
Transformer-KNN，不能把它写成 SVR 参数优化结果。

## 4. 修改和新增的代码

- `src/train.py`：增加 `--enable-feature-filtering` 和
  `--feature-missing-threshold` 命令行参数。
- `src/tune_svr.py`：冻结 Transformer 并调节普通 SVR；支持原始坐标目标。
- `src/tune_svr_fusion.py`：新增的特征融合 Transformer-SVR 正式脚本。
- `src/tune_knn.py`：新增的冻结 Transformer + KNN 正式脚本。
- `src/model.py`：未修改 Transformer 主体结构。

特征融合和 KNN 实验都读取以下已训练检查点，并冻结其权重：

```text
model/wap-gpu-building0-filter98-e80/20260721_193744
```

## 5. 正式复现命令

在项目根目录执行。

特征融合 Transformer-SVR：

```powershell
python -m src.tune_svr_fusion --model-dir model\wap-gpu-building0-filter98-e80\20260721_193744 --building-id 0 --device cuda --feature-batch-size 256 --folds 2 --cv-group position --c-values 50 100 200 --epsilon-values 0.1 0.3 0.5 --output-dir model\frozen-transformer-fusion-svr-filter98
```

Transformer-KNN：

```powershell
python -m src.tune_knn --model-dir model\wap-gpu-building0-filter98-e80\20260721_193744 --building-id 0 --device cuda --feature-batch-size 256 --folds 2 --cv-group phone --output-dir model\frozen-transformer-knn-filter98
```

## 8. 汇报时必须说明的实验规范

1. 最终报告 JSON 中的 `evaluation`，不是 `best_cross_validation`，也不是训练日志中的 Validation。
2. 参数选择只使用 trainingData.csv；validationData.csv 只用于最终 Evaluation。
3. 原论文的 6.51/4.79 应标注为“论文报告值”，不能假装是本地重新运行结果。

4. 特征融合方法仍是 Transformer-SVR；KNN 方法是 Transformer-KNN 下游头变体。

## 9. 一分钟口头汇报模板

原论文采用 Transformer 编码 WiFi 指纹，再由 SVR 回归经纬度。我们发现
Building 0 中大量 WAP 的缺失率超过 98%，因此先过滤无效 WAP，保留 98 个
有效特征。在不修改 Transformer 结构和权重的前提下，我们提出特征融合改进：
将 Transformer 输出的 64 维高层 latent 与 98 维过滤 RSSI 拼接，再输入
RBF-SVR。参数通过训练集位置分组交叉验证选择，最终平均误差为
6.594 米，中位误差为 4.958 米。进一步把
下游回归头替换为距离加权 KNN 后，平均误差达到 6.105 米，中位误差达到
4.359 米。两项改进均复用并冻结同一个 Transformer，未修改大模型主体。
